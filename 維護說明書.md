## 一、先搞懂三件事

| 名詞 | 白話解釋 |
| --- | --- |
| **GitHub** | 網站「原始檔案」存放的雲端資料夾，網址是 `github.com/jeremy0819/jieceng-web`。所有修改都從這裡開始。 |
| **main 分支** | 正式版本。**只要把修改存進 `main`，機器人就會自動更新網站。** |
| **GitHub Actions** | 自動化機器人。存檔後約 2～5 分鐘自動蓋好網站並上線，你什麼都不用做。 |

> ⚠️ **重要：倉庫必須維持「公開（Public）」狀態。**
> 若把倉庫改為私人（Private），GitHub Pages 會立即停止服務、網站變成 404。

## 二、網站自動上線流程

```
你在 GitHub 上修改文字／圖片（在 main 分支）
        ↓
按下「Commit（儲存）」到 main 分支
        ↓
機器人 (GitHub Actions) 自動觸發（約 2～5 分鐘）：
  → 安裝套件 → 編譯網站 → 推送到 gh-pages → 上線
        ↓
網站自動更新，完成！
```

**確認機器人有沒有做完**：到 GitHub 倉庫上方的 **Actions** 分頁，看到綠色 ✅ = 成功；紅色 ❌ = 有問題（見[第十一節](#十一常見問題與排除)）。

## 三、修改文字內容

> ✏️ **黃金原則：只改引號中間的文字，不要動引號、逗號、括號等符號。動到符號 = 網站壞掉。**

### 修改建案介紹文字

**目標檔案**：`composables/useProjects.ts`

1. 打開 `https://github.com/jeremy0819/jieceng-web`
2. 點進 `composables` 資料夾 → 點 `useProjects.ts`
3. 點右上角**鉛筆圖示 ✏️** 進入編輯模式
4. 用 `Ctrl+F` 搜尋你要改的建案名稱
5. **只改單引號 `'` 中間的文字**，例如：
   ```
   改前：description: '台灣首座六大標章認證豪宅。',
   改後：description: '台灣首座榮獲六大建築標章的精品住宅。',
   ```
6. 捲到頁面最下方 → **Commit changes** → Summary 填說明（例：`修改印象羅芙描述`）→ 選 **Commit directly to the main branch** → 按綠色 **Commit changes**

### 各頁面文字對應檔案

| 要改的頁面 | 檔案位置 |
| --- | --- |
| 首頁標語 | `pages/index.vue` |
| 關於我們（品牌故事、里程碑） | `pages/about.vue` |
| 全案管理 PCM | `pages/pcm.vue` |
| iHome 5.0 五大宅 | `pages/ihome.vue` |
| 建案精選集（頁面介紹文字） | `pages/portfolio.vue` |
| 都更說明會（說明文字） | `pages/briefing.vue` |
| 標章介紹 | `pages/presentations.vue` |
| 聯絡我們 | `pages/contact.vue` |
| iHome 五大宅詳細說明 | `composables/useIHome.ts` |
| 建案資料（標題、描述、亮點） | `composables/useProjects.ts` |

> ⚠️ `.vue` 檔裡的文字有時在 `>` 和 `<` 之間（例如 `<p>這是文字</p>`），只改標籤中間的文字，不要動 `<` `>`。

## 四、換圖片

所有圖片放在 **`public/image/`** 資料夾。

### 目前圖片清單

| 檔名 | 用途 |
| --- | --- |
| `hero-background.jpg` | 各頁面 Hero 背景、首頁大圖 |
| `project-1.jpg` | 印象羅芙 建案圖片 |
| `project-2.jpg` | 印象天裔 建案圖片 |
| `project-3.jpg` | 新店安和段 建案圖片 |
| `project-4.jpg` | 印象丞裔 建案圖片（兼用於關於我們頁大圖） |

### 換掉現有圖片（標準作業程序）

**原則：新圖片檔名要與舊圖片完全一致，上傳後自動覆蓋。**

1. **確認舊圖片檔名**：進入 `public/image/` 資料夾，記下要替換的檔名（例如 `project-1.jpg`）
2. **準備新圖片**：在電腦上把新圖片重新命名，確保與舊圖**完全一樣**（例如 `project-1.jpg`）
   - 命名規則：全英文小寫、不含中文、不含空格
   - 建議規格：寬度 1600px 以上，500KB 以內（可用 TinyPNG 壓縮）
3. **上傳圖片**：在 `public/image/` 資料夾頁面，點右上角 **`Add file` → `Upload files`**，拖入新圖
4. **儲存**：下方 Commit changes → Summary 填 `更換 project-1 圖片` → Commit to main
5. 等 2～5 分鐘，重新整理網站確認圖片更新

### 新增全新圖片（給新建案用）

1. 命名新圖片（例如 `project-5.jpg`）
2. 依上方步驟上傳到 `public/image/`
3. 在 `composables/useProjects.ts` 的新建案資料中，把 `image` 欄位改成新圖片路徑（見[第五節](#五新增一個建案)）

## 五、新增一個建案

> 💡 新增建案需要**同時修改三個地方**，缺一不可：圖片 → 建案資料 → 路由登記。

---

### 步驟一：上傳建案圖片

依照[第四節新增圖片流程](#新增全新圖片給新建案用)，上傳圖片到 `public/image/`。

**命名建議**：`project-5.jpg`（數字對應新建案的 id，見步驟二）

---

### 步驟二：新增建案資料

**目標檔案**：`composables/useProjects.ts`

1. 打開檔案，點鉛筆 ✏️ 進入編輯模式
2. 捲到**最底部**，在最後一個 `},` 後方新增：

```ts
  {
    id: 5,
    title: '傑丞。新建案名稱',
    year: 2027,
    category: 'residential',
    categoryLabel: '住宅建案',
    location: '桃園市桃園區',
    description: '一句話短介紹（約 20 字以內）。',
    fullDescription: '詳細完整介紹，可以多段。',
    image: 'image/project-5.jpg',
    highlights: [
      '亮點一',
      '亮點二',
      '亮點三',
      '亮點四',
    ],
    specs: {
      基地位置: '桃園市桃園區',
      完工年份: '2027',
      建築類型: '住宅',
      工法: 'iHome 5.0 全系統',
    },
    technologies: ['centennial', 'healthy', 'energy'],
  },
```

**欄位說明：**

| 欄位 | 說明 | 注意事項 |
| --- | --- | --- |
| `id` | 建案編號（數字） | 目前最大是 **4**，新的填 **5**，依此類推 |
| `title` | 建案名稱 | 慣例格式「傑丞。建案名稱」 |
| `year` | 完工年份 | **數字，不加引號**，例如 `2027` |
| `category` | 分類代碼 | `'residential'`（住宅建案）或 `'renewal'`（都更案） |
| `categoryLabel` | 分類中文名稱 | 對應 category：`'住宅建案'` 或 `'都更案'` |
| `image` | 圖片路徑 | 開頭**不加斜線**，格式：`image/project-4.jpg` |
| `highlights` | 亮點（固定 4 項） | 詳情頁 2×2 格版面，**必須填滿 4 項** |
| `technologies` | iHome 工法標籤 | 決定建案出現在哪些篩選結果（見下表） |

**`technologies` 可用代碼（iHome 五大宅）：**

| 代碼 | 中文 |
| --- | --- |
| `'centennial'` | 百年宅 |
| `'healthy'` | 健康宅 |
| `'energy'` | 節能宅 |
| `'smart'` | 智慧宅 |
| `'resume'` | 履歷宅 |

3. 改完後 Commit to main，填說明例如：`新增建案：傑丞新建案名稱`

---

### 步驟三：登記新頁面路由

**目標檔案**：`nuxt.config.ts`

> ⚠️ **容易忘記！漏掉這步，新建案的詳情頁部署後會出現 404。**

1. 打開 `nuxt.config.ts`，點鉛筆 ✏️
2. 用 `Ctrl+F` 找到 `'/project/4'`（目前最後一個）
3. 在它下方新增：`'/project/5',`

   ```
   改前：
       '/project/4'
     ]
   改後：
       '/project/4',
       '/project/5',
     ]
   ```

4. Commit to main，填說明：`登記 project 5 路由`

---

### 步驟四：確認

三個檔案都 Commit 後：
1. 到 **Actions** 分頁等待綠色 ✅（約 2～5 分鐘）
2. 打開 `https://jeremy0819.github.io/jieceng-web/portfolio` 確認新建案出現
3. 點進新建案詳情頁，確認不是 404

## 六、維護都更說明會場次

說明會場次資料位於 `pages/briefing.vue` 檔案的 `upcomingSessions` 陣列中。

**目標檔案**：`pages/briefing.vue`

### 新增一個場次

1. 打開 `pages/briefing.vue`，點鉛筆 ✏️
2. 用 `Ctrl+F` 找到 `upcomingSessions`
3. 在最後一個 `},` 後方加入新場次：

```js
  {
    month: 'SEP',       // 月份（英文縮寫）
    day: '20',          // 日期（字串）
    weekday: 'SAT',     // 星期（英文縮寫）
    type: '都更說明',   // 場次類型（'都更說明' 或 '新案說明'）
    title: '場次名稱',  // 說明會標題
    time: '14:00 – 16:00',
    location: '地點名稱',
    spots: 15,          // 剩餘名額（填數字；若不限填 null）
  },
```

4. Commit to main，填說明：`新增 9/20 都更說明會場次`

### 修改現有場次

直接修改對應場次的欄位文字，Commit to main 即可。

### 刪除已過期場次

把整個場次的 `{...},` 區塊刪除，注意確認前後的逗號正確，Commit to main。

**目標檔案**：`composables/useIHome.ts`

這個檔案控制五大宅的所有說明內容，首頁互動展示、iHome 專頁、導覽列下拉選單都從這裡讀取。

### 修改五大宅說明文字

1. 打開 `composables/useIHome.ts`，點鉛筆 ✏️
2. 用 `Ctrl+F` 找到要改的宅名稱（例如「百年宅」）
3. 只改引號中間的文字，例如：
   ```
   改前：subtitle: 'Open Building & SI Methodology',
   改後：subtitle: 'Centennial Durability & SI Method',
   ```

### ⚠️ 絕對不能改的欄位

`key` 欄位（`centennial` / `healthy` / `energy` / `smart` / `resume`）是全站識別工法的唯一代碼，**絕對不能修改**。一旦改動，所有建案的工法標籤和篩選功能都會失效。

## 十一、常見問題與排除

**Q1：網站打開是 404 / 空白頁？**

- [ ] 網址是否正確：`https://jeremy0819.github.io/jieceng-web/`（結尾的 `/jieceng-web/` 不能少）
- [ ] 到 **Actions** 分頁確認最新一筆是否為綠色 ✅（若是紅色 ❌，見 Q3）
- [ ] 到 `Settings → Pages` 確認 Source 是 `Deploy from a branch`，Branch 是 `gh-pages`，目錄是 `/ (root)`
- [ ] 等 5 分鐘後強制重新整理（`Ctrl+F5` / `Cmd+Shift+R`）

---

**Q2：改了文字但網站沒有更新？**

- 機器人需要 2～5 分鐘，先到 **Actions** 確認綠色 ✅
- 強制重新整理瀏覽器快取（`Ctrl+F5`）
- 確認你 Commit 到的是 `main` 而不是其他分支

---

**Q3：Actions 出現紅色 ❌，不知道原因？**

1. 點進紅色的工作流程記錄 → 點 **`deploy`** 步驟 → 看哪一行出錯
2. 最常見原因：修改 `.ts` 或 `.vue` 檔時，不小心刪掉或多加了符號（逗號、引號、括號）
3. 修正方法：把出問題的檔案內容複製，貼給 Claude Code 請它找錯

---

**Q4：我不小心改壞了，想還原？**

1. 進入出問題的檔案頁面
2. 點右上角 **`History`**（歷史記錄）
3. 找到改壞之前的那一筆 Commit，點進去
4. 把舊版本的內容複製出來，貼回現在的檔案，再 Commit

---

**Q5：倉庫改過「私人（Private）」，網站又壞了？**

1. 到 `Settings` → 最底部 Danger Zone，確認倉庫是 **Public**
2. 到 `Settings → Pages`，Source 選 **`Deploy from a branch`**，Branch 選 **`gh-pages`** / `/ (root)` → Save
3. 到 **Actions** 點最新一筆 → **`Re-run all jobs`** 重新觸發部署
4. 等 2～5 分鐘確認網站恢復

---

**Q6：都更說明會頁的 FAQ 展開不正常？**

`briefing.vue` 中的 FAQ 需要 `openRenewalFaq` 這個響應式變數正確設置（`const openRenewalFaq = ref(-1)`）。若修改 FAQ 時不小心刪除了這行，就會出問題，請確認這行存在於 `<script setup>` 區塊中。

## 十二、部署架構說明（給工程師）

本網站採用 **Nuxt 3 SSG** 架構，部署於 GitHub Pages。

**部署流程：**
1. Push 到 `main` 分支
2. GitHub Actions 觸發 `.github/workflows/deploy.yml`
3. 執行 `npm run generate`，產生靜態檔案至 `.output/public/`
4. 使用 `peaceiris/actions-gh-pages@v4` 推送到 `gh-pages` 分支
5. GitHub Pages 從 `gh-pages` 分支提供服務

**使用 peaceiris 而非 actions/deploy-pages 的原因：**
GitHub 原生方案依賴 `github-pages` Environment，在 Public ↔ Private 切換後憑證會被清空，導致靜默失敗。peaceiris 透過 git push 部署，不依賴 Environment，容錯性更強。

**子路徑設定：**
倉庫名稱為 `jieceng-web`，GitHub Pages 掛載於 `/jieceng-web/` 子路徑。部署腳本透過 `DEPLOY_BASE` 環境變數傳入 Nuxt，所有 `withBase()` 調用確保資源路徑正確。

**設計 Tokens：**

## 十三、維護黃金守則

1. **一次只改一個地方**，改完確認 Actions ✅ 後，再改下一個
2. **只改文字內容**，不要動符號（引號、逗號、括號）
3. **倉庫永遠保持 Public**，不要改成 Private
4. **新增建案必須同時完成三步**：圖片 → useProjects.ts → nuxt.config.ts 路由登記
5. **都更說明會場次過期後記得刪除**，以免網站顯示過期資訊
6. 不確定的時候，把檔案內容貼給 Claude Code 問它，比自己硬改安全得多

## 附錄：GitHub Pages 來源設定（首次啟用或恢復時）

> 正常情況下不需要動這裡。只有在網站無緣無故壞掉時，才來這裡確認一次。

1. 打開 `https://github.com/jeremy0819/jieceng-web/settings/pages`
2. 找到 **Source（來源）** 區塊
3. 確認下拉選單顯示 **`Deploy from a branch`**（若顯示 `GitHub Actions`，請改回此選項）
4. Branch 欄位選 **`gh-pages`**，目錄選 **`/ (root)`**
5. 按 **Save**，等待 1～2 分鐘重新整理網站

## 附錄：GitHub 倉庫所有權轉移 SOP

當需要把倉庫從一個 GitHub 帳號轉移到另一個帳號時，務必使用「正式轉移所有權」，**不要用 Fork**。

### 步驟 1：從原帳號發起轉移

1. 用原帳號登入，進入 `github.com/原帳號/jieceng-web`
2. **Settings → General → 最下方 Danger Zone → Transfer ownership**
3. 填入新帳號名稱，確認 → 送出

### 步驟 2：新帳號接受轉移

1. 用新帳號登入，接受 GitHub 通知的轉移邀請

### 步驟 3：重新開啟 GitHub Pages

1. 進入新帳號的 `jieceng-web` → **Settings → Pages**
2. Source：**Deploy from a branch**，Branch：**`gh-pages`** / **`/ (root)`** → Save

### 步驟 4：確認 Actions 仍運作

進 **Actions** 分頁，若看到提示就點 **Enable workflows**，之後 push 到 `main` 即可觸發。

> **重要提醒**：倉庫名稱若維持 `jieceng-web`，子路徑不變，轉移後網站立即上線，不需要修改任何程式碼。
