## 一、先搞懂三件事

| 名詞 | 白話解釋 |

| **GitHub** | 網站「原始檔案」存放的雲端資料夾，網址是 `github.com/jeremy0819/jieceng-web`。所有修改都從這裡開始。 |

## 二、網站怎麼自動上線的

```
你在 GitHub 上修改文字／圖片
        ↓
按下「Commit（儲存）」到 main 分支
        ↓
機器人 (GitHub Actions) 自動開始重蓋網站（約 2～5 分鐘）
        ↓
網站自動更新，完成！
```

你**不需要**手動「發佈」或「部署」。存檔 = 上線。

> 想看機器人有沒有做完？到 GitHub 倉庫上方的 **Actions** 分頁，看到綠色勾勾 ✓ 就代表成功了；如果是紅色叉叉 ✗，代表有問題，請看[第八節](#八常見問題與排除)。

## 三、日常維護：修改文字

以「修改某個作品的介紹文字」為例：

1. 打開 `https://github.com/jeremy0819/jieceng-web`
2. 點進 `composables` 資料夾 → 點 `useProjects.ts` 檔案
3. 點右上角的**鉛筆圖示** ✏️（Edit this file）
4. 找到你要改的文字，直接修改。例如：
   ```
   title: '開放建築',          ← 作品名稱
   description: '利用自然採光…', ← 卡片上的短介紹
   fullDescription: '…',       ← 點進去後的完整介紹
   ```
   ⚠️ **只改引號 `'` 中間的文字**，不要動到引號、逗號或其他符號。
5. 改好後，捲到頁面最下方，在 **Commit changes** 區塊：
   - 第一格填一句說明，例如「修改開放建築的介紹文字」
   - 確認選擇 **Commit directly to the `main` branch**
   - 按綠色按鈕 **Commit changes**
6. 等 2～5 分鐘，重新整理網站就會看到更新。

> 首頁、關於我們等頁面的文字，分別在 `pages` 資料夾裡的各個檔案（`index.vue`、`about.vue` 等），改法完全一樣：找到中文字、只改引號或標籤中間的文字。

## 四、日常維護：換圖片

所有圖片都放在 `public/image/` 資料夾。

**換掉現有圖片（最簡單）**：用同樣的檔名覆蓋即可，網站會自動換新。

1. 進入 `public/image/` 資料夾
2. 按右上角 **Add file → Upload files**
3. 把新圖片拖進去。**檔名要和舊的完全一樣**（例如舊的叫 `project-1.jpg`，新的也要命名為 `project-1.jpg`）
4. 下方 **Commit changes** → 綠色按鈕儲存

**圖片注意事項**：
- 檔名一律用**英文小寫**，不要有中文或空格（例如 `project-5.jpg` ✓、`作品五.jpg` ✗）
- 建議寬度 1600px 以上，檔案盡量壓到 500KB 以內，網站才會載得快

## 五、日常維護：新增一個作品

新增作品需要改**三個地方**，缺一不可。最簡單的做法是直接請 Claude 幫你做（見[第七節](#七用-claude-來幫你維護含-skill-管理)），它已經內建了「新增作品」的自動流程。若要手動操作：

**步驟 1：上傳圖片**
把新作品圖片上傳到 `public/image/`，命名例如 `project-5.jpg`（英文小寫）。

**步驟 2：新增作品資料**
編輯 `composables/useProjects.ts`，在作品清單最後（最後一個 `}` 後面）加一段：
```
  {
    id: 5,
    title: '新作品名稱',
    category: 'Residential',
    description: '一句話短介紹（卡片上顯示）。',
    fullDescription: '完整的詳細介紹（點進去後顯示）。',
    image: 'image/project-5.jpg',
    specs: {
      Location: '台北市, 信義區',
      Year: '2026',
      Area: '120 sqm',
      System: '工法名稱',
    },
  },
```
- `id` 要用目前最大的數字 +1（目前最大是 4，所以新的填 5）
- `category` 只能填這五個英文其中之一：`Residential`（住宅）、`Commercial`（商業）、`Landscape`（景觀）、`Renovation`（改造）

**步驟 3：登記新頁面**
編輯 `nuxt.config.ts`，找到 `routes:` 那段清單，加一行 `'/project/5',`（數字對應你的新 id）。
⚠️ 漏掉這步，新作品的頁面上線後會出現 404。

三個檔案都 Commit 到 `main` 後，等幾分鐘即可上線。

## 七、用 Claude 來幫你維護（含 Skill 管理）

你不必自己記住上面的步驟，可以直接用白話請 **Claude** 幫忙，例如：
- 「幫我把『開放建築』的介紹改成 ⋯⋯」
- 「我要新增一個作品，叫『山居歲月』，住宅類，圖片我等等上傳」

### 什麼是 Skill（技能）？

**Skill 就是預先寫好的一份「工作步驟說明書」**，存成檔案放在專案裡。Claude 看到相關需求時，會自動照著這份說明書執行，確保每次都按正確流程、不會漏步驟。

本專案已經內建了一個 Skill：**`add-project`（新增作品）**。當你說「新增作品」時，Claude 會自動載入它，照著「上傳圖片 → 改資料 → 登記路由 → 驗證」的標準流程幫你做完。

### Skill 存在哪裡？

```
.claude/
└── skills/
    └── add-project/
        └── SKILL.md   ← 技能的內容（步驟說明）寫在這裡
```

每一個 Skill 就是一個資料夾，裡面放一個 `SKILL.md` 檔案。

### 如何「管理 / 儲存 / 修改」Skill

**① 查看現有的 Skill**
進入 GitHub 的 `.claude/skills/` 資料夾，每個子資料夾就是一個技能。

**② 修改現有 Skill**
打開該技能的 `SKILL.md`，點鉛筆 ✏️ 編輯文字（就是改步驟說明），Commit 到 `main` 即可。

**③ 新增一個 Skill**
1. 在 `.claude/skills/` 底下，用 **Add file → Create new file** 建立檔案
2. 檔名輸入：`你的技能名稱/SKILL.md`（打斜線 GitHub 會自動建立資料夾）
3. 檔案最上方一定要有這段「標頭」，Claude 才認得：
   ```
   ---
   name: 技能英文代號
   description: 一句話說明這個技能在做什麼、使用者說什麼時候要用它。
   ---
   ```
4. 標頭下面用中文寫清楚「要照什麼步驟做」即可。
5. Commit 到 `main`。

**④ 刪除 Skill**
打開該技能的 `SKILL.md`，編輯畫面右上角選 **Delete file**，Commit 即可。

> 小提醒：`description` 寫得越清楚（包含「使用者會怎麼開口要求」），Claude 越能在對的時機自動使用這個技能。

## 八、常見問題與排除

**Q1：網站打開是 404 / 空白頁？**
- 先確認網址是否正確：`https://jeremy0819.github.io/jieceng-web/`（結尾的 `/jieceng-web/` 不能少）
- 到 GitHub 倉庫的 **Actions** 分頁，看最新一筆是不是紅色叉叉 ✗。若是，點進去看哪一步失敗，通常是改檔案時不小心刪到符號（逗號、引號、括號）。把該檔案改回正確格式即可。
- 第一次部署時，需到倉庫 **Settings → Pages**，確認 **Source** 選的是 **GitHub Actions**。

**Q2：改了文字但網站沒變？**
- 機器人需要 2～5 分鐘。請先到 Actions 分頁確認有綠色勾勾 ✓。
- 瀏覽器可能有快取，按 `Ctrl + F5`（Mac 是 `Cmd + Shift + R`）強制重新整理。

**Q3：Actions 出現紅色叉叉，看不懂錯誤？**
- 最常見原因是改 `useProjects.ts` 或 `nuxt.config.ts` 時少打或多打了符號。
- 最保險的做法：把剛剛的修改撤回，或直接把檔案內容貼給 Claude，請它幫你找出哪裡符號錯了。

**Q4：我不小心改壞了，想還原？**
- 在 GitHub 該檔案頁面點 **History（歷史）**，找到改壞之前的版本，可以還原。
- 或直接告訴 Claude「幫我把某檔案還原到上一個正常版本」。
